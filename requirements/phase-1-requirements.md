# 🎨 图片生成前端代码 - 第一期需求文档

## 项目名称
**Image2Code** - 图片转前端代码生成平台

## 阶段目标
实现最核心的图片到代码生成预览功能，建立多Agent协作框架的最小可用系统（MVP）。

---

## 一、核心功能范围

### 1.1 必须实现功能

| 序号 | 功能模块 | 功能描述 | 优先级 |
|------|----------|----------|--------|
| 1 | 图片上传 | 支持单张图片上传（PNG/JPG/WebP） | P0 |
| 2 | 图片分析 | 分析Agent理解图片内容，提取UI元素 | P0 |
| 3 | 代码生成 | Coder Agent根据分析结果生成前端代码 | P0 |
| 4 | 代码预览 | iframe实时预览生成的前端代码 | P0 |
| 5 | 简单聊天 | 用户可与系统对话，说明生成需求 | P0 |

### 1.2 简化版Agent架构

```
用户上传图片 + 需求描述
        ↓
   [Coordinator]
        ↓
   [Analyzer] → 分析图片，提取UI元素和布局
        ↓
   [Coder] → 生成HTML/CSS代码
        ↓
   [Preview] → 代码预览
```

### 1.3 不包含功能（后续迭代）
- 多张图片上传
- 多种框架选择（React/Vue）
- 代码编辑功能
- 代码审核
- 项目导出
- 历史记录
- 记忆服务

### 2.2 分析Agent模块

#### 功能需求
- [ ] 接收用户上传的图片
- [ ] 调用LLM分析图片内容
- [ ] 提取以下信息：
  - 页面布局结构（头部、侧边栏、主内容、底部）
  - UI组件类型（按钮、输入框、卡片、列表、表格等）
  - 颜色方案（主色、背景色、强调色）
  - 字体风格
  - 间距和尺寸
- [ ] 返回结构化的分析结果

#### 输出格式
```typescript
interface AnalysisResult {
  layout: {
    type: 'single-column' | 'two-column' | 'grid' | 'hero';
    sections: string[];
  };
  components: {
    name: string;
    type: 'button' | 'input' | 'card' | 'navbar' | 'footer' | 'list' | 'table';
    properties: Record<string, any>;
  }[];
  colors: {
    primary: string;
    secondary: string;
    background: string;
    text: string;
  };
  style: {
    borderRadius: string;
    shadow: string;
    spacing: string;
  };
}
```

### 2.3 代码生成Agent模块

#### 功能需求
- [ ] 接收分析结果和用户需求描述
- [ ] 生成单一HTML文件（内联CSS）
- [ ] 代码要求：
  - 使用现代CSS（Flexbox/Grid）
  - 响应式设计（基础适配）
  - 清晰的代码结构
  - 适当的注释
- [ ] 支持以下类型页面生成：
  - 登录/注册页面
  - 仪表盘首页
  - 列表页面
  - 表单页面
  - 卡片展示页面
  - 通用落地页
- [ ] 语法错误检测和基础修复

#### 输出格式
```html
<!DOCTYPE html>
<html>
<head>
  <style>
    /* 内联CSS样式 */
  </style>
</head>
<body>
  <!-- HTML结构 -->
</body>
<script>
  <!-- 基础交互脚本 -->
</script>
</html>
```

## 五、Prompt工程设计

### 5.1 Analyzer Agent Prompt

```
你是一个专业的UI/UX分析师。请分析用户上传的图片中的界面设计。

请提取以下信息并以JSON格式返回：
1. 页面布局类型（单栏/双栏/网格/英雄区）
2. 包含的UI组件列表（按钮、输入框、卡片、导航栏等）
3. 配色方案（主色、背景色、文字色、强调色）
4. 字体风格（字号、字重）
5. 间距和圆角风格
6. 整体页面类型（登录页、仪表盘、列表页等）

请只返回JSON，不要其他内容。
```

### 5.2 Coder Agent Prompt

```
你是一个专业的前端开发者。请根据以下分析结果生成HTML代码。

分析结果：
{analysisResult}

用户需求：{userDescription}

要求：
1. 生成单一HTML文件，包含内联CSS
2. 使用现代CSS布局（Flexbox/Grid）
3. 代码简洁、语义化
4. 保持与原图相似的视觉效果
5. 添加适当的交互效果（hover、focus等）
6. 确保代码可以直接在浏览器中运行

请只返回HTML代码，不要其他内容。
```

## 六、验收标准

### 6.1 功能验收

| 验收项 | 验收标准 |

## 八、风险与限制

### 8.1 已知限制

- 仅支持单张图片上传
- 仅支持HTML+CSS输出（不支持React/Vue）
- 不支持复杂交互（仅基础hover效果）
- 图片识别准确率依赖LLM能力
- 生成的代码可能需要手动调整

### 8.2 潜在风险

- LLM生成代码可能存在语法错误
- 复杂页面布局可能生成不完整
- 预览可能存在跨域问题
