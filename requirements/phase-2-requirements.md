# 🎨 图片生成前端代码 - 第二期需求文档

## 项目名称
**Image2Code** - 图片转前端代码生成平台

## 阶段目标
在第一期基础上，增加代码编辑、代码审核、多轮优化功能，提供更完善的代码生成体验。

---

## 一、本期新增功能概览

### 1.1 功能清单

| 序号 | 功能模块 | 功能描述 | 优先级 |
|------|----------|----------|--------|
| 1 | 代码编辑器 | Monaco Editor在线编辑代码 | P0 |
| 2 | 代码审核 | Reviewer Agent自动审核代码 | P0 |
| 3 | 多轮优化 | 用户反馈后重新生成代码 | P0 |
| 4 | 代码格式化 | Prettier自动格式化代码 | P1 |
| 5 | 语法检测 | 生成后自动检测语法错误 | P1 |

### 1.2 保留第一期功能
- ✅ 图片上传与分析
- ✅ 代码生成
- ✅ 实时预览
- ✅ 简单聊天

## 二、详细功能需求

### 2.1 代码编辑器模块

#### 功能需求
- [ ] 集成 Monaco Editor 代码编辑器
- [ ] 支持 HTML/CSS/JS 语法高亮
- [ ] 支持代码自动补全

- [ ] 支持代码搜索和替换
- [ ] 主题切换（浅色/深色）
- [ ] 支持撤销/重做

### 2.2 代码审核模块

#### 功能需求
- [ ] 自动检测代码语法错误
- [ ] 检测HTML语义化问题
- [ ] 检测CSS问题（选择器冲突、属性废弃等）
- [ ] 检测JavaScript潜在问题
- [ ] 生成审核报告
- [ ] 提供修复建议

#### 审核规则
```typescript
interface ReviewResult {
  errors: {
    line: number;
    severity: 'error' | 'warning' | 'info';
    message: string;
    suggestion?: string;
  }[];
  score: number;  // 0-100
  summary: string;
}
```

#### Reviewer Agent Prompt
```
你是一个代码审查专家。请审核以下HTML代码中的问题。

代码：
{code}

请检查以下方面：
1. HTML语法错误
2. CSS错误或最佳实践
3. JavaScript错误
4. 可访问性问题
5. 性能问题

请以JSON格式返回审核结果：
{
  "errors": [{ "line": 1, "severity": "error", "message": "...", "suggestion": "..." }],
  "score": 85,
  "summary": "总体评价"
}
```

### 2.3 多轮优化模块

#### 功能需求
- [ ] 接收用户修改指令
- [ ] 理解自然语言修改需求
- [ ] 局部更新代码（不重新生成全部）
- [ ] 支持多次迭代优化
- [ ] 记录修改历史
- [ ] 支持版本回滚

#### 优化示例
```
用户: 按钮改成圆角
系统: 好的，正在修改...

用户: 背景色改成渐变
系统: 好的，正在修改...

用户: 恢复到第一版
系统: 已恢复到初始版本
```

#### 修改指令类型
- 样式修改：颜色、尺寸、间距、圆角、阴影
- 结构修改：添加组件、删除组件、调整布局
- 交互修改：添加动画、修改hover效果

### 2.4 代码格式化模块

#### 功能需求
- [ ] 一键格式化代码
- [ ] 自动缩进
- [ ] 换行整理
- [ ] CSS属性排序
- [ ] 保存时自动格式化（可选）

### 2.5 语法检测模块

#### 功能需求
- [ ] 生成代码后自动检测
- [ ] 实时编辑时检测（防抖）
- [ ] 高亮显示错误位置
- [ ] 提供快速修复建议

## 三、技术架构

### 3.1 扩展的Agent架构

```
                    ┌─────────────────┐
                    │   Coordinator   │
                    └────────┬────────┘
                             │
          ┌──────────────────┼──────────────────┐
          │                  │                  │
          ▼                  ▼                  ▼
   ┌────────────┐    ┌────────────┐    ┌────────────┐
   │  Analyzer  │    │   Coder     │    │  Reviewer  │
   │   Agent    │───▶│   Agent     │───▶│   Agent    │
   └────────────┘    └────────────┘    └────────────┘
                             │                  │
                             ▼                  ▼
                      ┌────────────┐    ┌────────────┐
                      │  Preview   │    │  Optimizer  │
                      │  Service   │    │   Agent    │
                      └────────────┘    └────────────┘
```

### 3.2 新增文件

```
server/src/services/agents/
├── coordinator.ts         # 更新：支持多轮优化
├── analyzer.ts           # 第一期已有
├── coder.ts              # 第一期已有
├── reviewer.ts           # 新增：代码审核
├── optimizer.ts          # 新增：代码优化
└── types.ts              # 更新：新增类型
```

### 3.3 API扩展

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/generate/optimize` | 优化代码 |
| POST | `/api/generate/review` | 审核代码 |
| POST | `/api/code/format` | 格式化代码 |
| GET | `/api/generate/history/:taskId` | 获取修改历史 |
| POST | `/api/generate/rollback/:taskId` | 回滚版本 |

## 四、用户体验改进

### 4.1 优化点

| 改进项 | 第一期 | 第二期 |
|--------|--------|--------|
| 代码查看 | 纯文本展示 | Monaco Editor编辑 |
| 代码质量 | 无审核 | 自动审核并修复 |
| 修改方式 | 重新生成 | 增量修改 |
| 版本管理 | 无 | 支持回滚 |
| 格式化 | 无 | 自动格式化 |

### 4.2 交互流程优化

```
第一期流程：
上传图片 → 生成代码 → 预览 → 结束

第二期流程：
上传图片 → 生成代码 → 审核(可选) → 预览 → 用户反馈 → 优化代码 → 预览 → 审核 → ... → 结束
```

## 五、验收标准

### 5.1 功能验收

| 验收项 | 验收标准 |
|--------|----------|
| Monaco Editor | 代码编辑流畅，语法高亮正确 |
| 代码审核 | 能检测并报告常见代码问题 |
| 多轮优化 | 能根据用户反馈修改代码 |
| 代码格式化 | 点击按钮后代码格式整洁 |
| 版本回滚 | 能恢复到历史版本 |

### 5.2 性能验收

| 指标 | 目标 |
|------|------|
| 编辑器响应 | ≤100ms |
| 代码审核 | ≤5秒 |
| 代码优化 | ≤10秒 |
| 格式化 | ≤2秒 |
